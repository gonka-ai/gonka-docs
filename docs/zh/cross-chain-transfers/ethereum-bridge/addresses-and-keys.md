# 地址与密钥

这是在进行跨链操作前最需要理解的关键页面。**请在首次转账前仔细阅读。**

## 一把私钥，两个地址

以太坊（Ethereum）和 Gonka 均使用相同类型的加密密钥（`secp256k1` 密钥对）。因此，**一个私钥可以同时控制两条链上的账户**。两者唯一的区别在于如何将公钥转换为人类可读的地址：

| 链 | 地址格式 | 地址从公钥推导的方式 |
| --- | --- | --- |
| 以太坊 | `0x...`（20 字节，十六进制） | `keccak256(未压缩公钥)` → 取最后 20 字节 |
| Gonka | `gonka1...`（bech32 编码） | `ripemd160(sha256(压缩公钥))` → 使用前缀 `gonka` 进行 bech32 编码 |

因此，同一个私钥会生成**两个外观不同的地址**——一个 `0x…` 地址和一个 `gonka1…` 地址——但这两个地址都由同一把私钥控制。

## 跨链桥如何决定代币的去向

当你将代币从以太坊跨链到 Gonka 时，你会将代币发送至桥接合约，并使用你的以太坊私钥签署这笔交易。Gonka 的跨链桥会：

1. 检测到以太坊上已完成的存款交易；
2. **从交易签名中恢复出公钥**；
3. 使用该公钥计算对应的 **Gonka 地址**（即上述 `gonka1…` 的标准推导方式）；
4. 将跨链代币铸造并发送至该 `gonka1…` 地址。

换句话说：**被封装的代币将被发送到由签署以太坊存款交易的同一公钥所生成的 Gonka 地址上**，而这个地址由与以太坊相同的私钥控制。要使用这些代币，你必须在 Gonka 上使用同一把私钥。

反向操作则有所不同：当你从 Gonka **跨链回以太坊**时，你可以在提现交易中**显式指定目标地址**；而当你**跨链到 Gonka 时，无法选择接收地址**——它由你的密钥唯一确定。

## 种子短语的陷阱（请务必阅读！）

大多数用户从未直接接触过原始私钥，而是通过一个**助记词（种子短语）**，由钱包自动从中派生出密钥。这种方式虽然方便，但在跨链时容易造成误解：

一个助记词并不对应单一密钥。钱包通过 **BIP-44 派生路径**从助记词生成密钥，而不同区块链使用**不同的路径**：

* 以太坊钱包使用币种类型 **60** → 路径为 `m/44'/60'/0'/0/0`
* Cosmos/Gonka 钱包使用币种类型 **118** → 路径为 `m/44'/118'/0'/0/0`

由于路径不同，**同一个助记词会为以太坊和 Gonka 生成完全不同的私钥**，进而生成两个毫无关联的地址。如果你从一个由助记词派生的以太坊账户发起存款，然后在钱包中查看由同一助记词派生的 Gonka 账户，**这两个账户并非同一把私钥**，因此跨链的代币将被发送到你的钱包并未显示、甚至可能无法控制的 `gonka1…` 地址上。

!!! danger
    不要认为“同一个助记词 = 两条链上的同一个账户”。跨链桥要求的是**在两条链上使用同一把私钥**，而不是同一个助记词。使用标准的不同派生路径会导致资金被发送到由你的以太坊私钥推导出的 Gonka 地址上——而你的 Gonka 钱包因使用不同路径派生，无法控制该地址。这并非永久性丢失，你仍可从同一助记词中派生出原始的以太坊私钥，该私钥即可控制 Gonka 链上的接收账户。

## 如何获取匹配的 Gonka 地址

你有两种选择。

### 选项 A — 使用仪表盘（推荐）

Gonka 仪表盘可自动解决密钥派生问题。只需使用你进行跨链操作的**同一个以太坊钱包连接仪表盘**，它便会自动计算并显示跨链桥将使用的正确 `gonka1…` 地址，展示你的封装代币余额，并引导你完成存款和提现流程。此方式无需手动处理私钥，也避免了上述助记词派生混淆的问题。

访问仪表盘地址：

```text
https://node1.gonka.ai:8443/dashboard/
```

### 选项 B — 将相同的私钥导入 Gonka 密钥环

如果你使用命令行操作，请将控制你以太坊账户的**完全相同**的 `secp256k1` 私钥（十六进制格式）导入到 Gonka 密钥环中。生成的 `gonka1…` 地址即为跨链桥铸币的目标地址：

```bash
inferenced keys import-hex <key_name> <YOUR_PRIVATE_KEY_HEX>

# Show the derived Gonka address
inferenced keys show <key_name> -a
```

此处打印的地址正是将接收您跨链代币的 `gonka1…` 地址，该密钥可为其签署 Gonka 上的交易（转账、提现等）。

!!! warning
    导入原始私钥会将其暴露于您导入的设备和密钥环中。建议在安全设备上使用基于文件的密钥环（`--keyring-backend file`），切勿将同时保护大量以太坊资产的私钥粘贴到不受信任的主机上。如有疑问，请使用仪表板（dashboard）操作。

## 快速检查清单

* 在开始之前，确定您将使用哪个密钥进行跨链操作。
* 如果您的以太坊地址上已有 USDT/ETH：从该密钥推导出对应的 `gonka1…` 地址（通过仪表板或 `import-hex` 命令）。
* 如果您想使用现有的 Gonka 地址：从该密钥推导出对应的 `0x…` 以太坊地址，并向其充值相应代币及足够的 ETH 作为燃料费。
* 务必先发送少量测试金额，并确认其已到达预期的 `gonka1…` 地址。
