# 地址和密钥

这是在跨链之前最需要理解的页面。**首次转账前请务必阅读。**

## 一个密钥，两个地址

以太坊和Gonka都使用相同类型的加密密钥（一个`secp256k1`密钥对）。因此，单个私钥控制着**两条链**上的账户。两条链的差异仅在于如何将公钥转换为人类可读的地址：

| 链 | 地址格式 | 地址如何从公钥派生 |
| --- | --- | --- |
| 以太坊 | `0x...` |  |
| (20字节，十六进制) |  |  |

因此，单个私钥会生成**两个外观不同的地址** — 一个是`0x…`，另一个是`gonka1…` — 但都由同一个密钥控制。

## 桥接器如何决定你的代币去向

当你将以太坊上的代币跨链到Gonka时，你会将它们发送到桥接合约，并使用你的以太坊私钥签署该以太坊交易。Gonka桥接器会：

1. 检测以太坊上已确认的存款交易。
2. **从交易签名中恢复公钥**。
3. 为该公钥计算**Gonka地址**（即上述的`gonka1…`标准派生方式）。
4. 将跨链代币铸造/释放到该`gonka1…`地址。

换句话说：**被封装的代币会被发送到由同一公钥生成的Gonka地址 — 也就是签署以太坊存款交易的同一个私钥所控制的地址。** 要使用这些代币，你必须在Gonka上使用相同的密钥。

反向操作则相反：当你**从Gonka跨链回以太坊**时，你可以在取款交易中**显式指定目标地址**；而当你**跨链到Gonka**时，你无法选择接收地址 — 它由你的密钥固定决定。

## 助记词陷阱（请阅读！）

大多数用户从未见过他们的原始私钥 — 他们只有**助记词**（助记短语），并让钱包为他们派生密钥。这虽然方便，但在跨链时会带来陷阱：

一个助记词并不对应单一密钥。钱包使用**BIP-44派生路径**从助记词派生密钥，而每条区块链使用**不同的路径**：

* 以太坊钱包使用币种类型**60** → 路径`m/44'/60'/0'/0/0`
* Cosmos/Gonka钱包使用币种类型**118** → 路径`m/44'/118'/0'/0/0`

由于路径不同，**同一个助记词会为以太坊和Gonka生成完全不同的私钥** — 因此地址也互不相关。如果你从一个由助记词派生的以太坊账户存款，然后在钱包中查看由助记词派生的Gonka账户，**它们并不是同一个密钥**，因此跨链的代币会到达一个你的钱包当前未显示给你的`gonka1…`地址。你仍然控制该地址（你可以使用相同的助记词并通过币种类型`60`派生以太坊密钥并在Gonka上使用），但你的钱包不会在没有手动派生步骤的情况下显示它。

!!! danger 
    **不要**假设“相同的助记词 = 两条链上的相同账户”。对于桥接，你需要的是两条链上**相同的私钥**，而不是相同的助记词。使用标准的不同派生路径会将你的资金发送到一个由你的以太坊密钥派生的地址 — 而你的Gonka钱包因使用不同的派生路径，并不控制该地址。这并非资金丢失，你仍然可以从相同的助记词派生出你的以太坊私钥，该密钥控制着Gonka链上的接收账户。

## 如何获取匹配的Gonka地址

你有两个选择。

### 选项A — 使用仪表板（推荐）

Gonka仪表板会为你解决派生问题。使用你进行跨链的**同一个以太坊钱包**连接，仪表板会自动派生并显示桥接器将使用的正确`gonka1…`地址，显示你的封装代币余额，并引导你完成存款/取款流程。这避免了你自己处理原始私钥，也消除了上述助记词混淆问题。

在以下地址打开仪表板：

```text
https://node1.gonka.ai:8443/dashboard/
```

### 选项 B — 将相同的私钥导入 Gonka 密钥环

如果你使用 CLI 操作，请将控制你以太坊账户的**完全相同**的 `secp256k1` 私钥（十六进制）导入 Gonka 密钥环。生成的 `gonka1…` 地址即为桥接合约铸币的目标地址：

```bash
inferenced keys import-hex <key_name> <YOUR_PRIVATE_KEY_HEX>

# Show the derived Gonka address
inferenced keys show <key_name> -a
```

此处打印的地址正是将接收您桥接代币的`gonka1…`地址，且该私钥可为Gonka交易（转账、提现）进行签名。

!!! warning 
    导入原始私钥会将该私钥暴露给您导入的设备和密钥环。建议在安全设备上使用基于文件的密钥环（`--keyring-backend file`），切勿在不受信任的主机上粘贴保护大量以太坊资金的私钥。如有疑问，请使用仪表板。
