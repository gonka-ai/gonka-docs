# 地址与密钥

在进行桥接前，这是最重要的一页，必须阅读。**首次转账前请务必阅读。**

## 一个密钥，两个地址

以太坊和 Gonka 都使用同一种加密密钥（`secp256k1` 密钥对）。因此，单个私钥可控制**两条链**上的账户。两条链的区别仅在于如何将公钥转换为人类可读的地址：

| 链 | 地址格式 | 地址如何从公钥推导 |
| --- | --- | --- |
| 以太坊 | `0x...`（20 字节，十六进制） | `keccak256(uncompressed_public_key)` → 最后 20 字节 |
| Gonka | `gonka1...`（bech32） | `ripemd160(sha256(compressed_public_key))` → 带 `gonka` 前缀的 bech32 |

因此，单个私钥会生成**两个外观不同的地址**——一个 `0x…` 和一个 `gonka1…`——但两者均由同一私钥控制。

## 桥接如何决定你的代币去向

当你将代币从以太坊桥接到 Gonka 时，你将代币发送至桥接合约，并使用你的以太坊私钥签署该以太坊交易。Gonka 桥接会：

1. 检测以太坊上已确认的存款交易。
2. **从你的交易签名中恢复公钥**。
3. 计算该公钥的**Gonka 地址**（如上所述的 `gonka1…` 标准推导方式）。
4. 铸造/释放桥接代币至该 `gonka1…` 地址。

换句话说：**封装代币将发送至由与签署以太坊存款交易相同的公钥生成的 Gonka 地址——且由同一私钥控制**。要花费这些代币，你必须在 Gonka 上使用相同的密钥。

反向操作则相反：桥接**回以太坊**时，你必须在提款交易中**显式指定目标地址**；而桥接**至 Gonka** 时，你无法选择收款方——地址由你的密钥固定。

## 助记词陷阱（请阅读！）

大多数用户从未见过原始私钥——他们仅拥有**助记词**（助记短语），并由钱包为其派生密钥。这很方便，但为桥接制造了一个陷阱：

助记词并不对应单个密钥。钱包使用**BIP-44 派生路径**从助记词派生密钥，而每条区块链使用**不同的路径**：

* 以太坊钱包使用币种类型 **60** → 路径 `m/44'/60'/0'/0/0`
* Cosmos/Gonka 钱包使用币种类型 **1200** → 路径 `m/44'/1200'/0'/0/0`

由于路径不同，**相同的助记词会为以太坊和 Gonka 生成两个完全不同的私钥**——因此产生两个无关的地址。如果你从助记词派生的以太坊账户存款，然后查看钱包中助记词派生的 Gonka 账户，**它们并非同一密钥**，因此桥接代币将存入一个你的钱包当前未显示的 `gonka1…` 地址。你仍然控制该地址（你可以使用币种类型 `60` 从同一助记词派生出以太坊私钥，并在 Gonka 上使用它），但你的钱包在未手动派生前不会显示该地址。

!!! danger 
    请**不要**假设“相同助记词 = 两条链上同一账户”。对于桥接，你需要的是**两条链上相同的私钥**，而非相同的助记词。使用标准的不同派生路径，会将你的资金发送至由你的以太坊密钥派生的地址——而你的 Gonka 钱包因使用不同路径派生，无法控制该地址。这并非资金损失，你仍可从同一助记词派生出以太坊私钥，该密钥控制着 Gonka 链上的接收账户。

## 如何获取匹配的 Gonka 地址

你有两种选择。

### 选项 A — 使用仪表板（推荐）

Gonka 仪表板为你自动处理派生。连接你用于桥接的**同一以太坊钱包**，仪表板将派生并显示桥接将使用的正确 `gonka1…` 地址，展示你的封装余额，并引导存款/提款流程。这避免了你直接处理原始私钥，并消除了上述助记词混淆问题。

打开仪表板：

```text
https://node1.gonka.ai:8443/dashboard/
```

### 选项 B — 将相同的私钥导入 Gonka 钱包密钥环

如果你使用 CLI，将控制你以太坊账户的**完全相同的** `secp256k1` 私钥（十六进制）导入 Gonka 钱包密钥环。生成的 `gonka1…` 地址即为桥接铸造代币的目标地址：

```bash
inferenced keys import-hex <key_name> <YOUR_PRIVATE_KEY_HEX>

# Show the derived Gonka address
inferenced keys show <key_name> -a
```

此处显示的地址正是将接收您跨链代币的 `gonka1…` 地址，该密钥可为其签署 Gonka 交易（转账、提现）。

!!! warning 
    导入原始私钥会将其暴露在您导入的机器和密钥环上。建议在安全的机器上使用基于文件的密钥环（`--keyring-backend file`），切勿将同时保护重要以太坊资金的私钥粘贴到不可信主机上。如有疑虑，请使用仪表板。
