# 多签签名者指南

本指南适用于被选为 Gonka 以太坊桥多签（Multisig）签名者的个人。

## 什么是 Safe 多签？
**Safe**（原 Gnosis Safe）是一种运行在以太坊区块链上的**智能合约账户**。与普通钱包（仅有一个私钥）不同，多签需要多个授权签名者共同批准交易后才能执行。

在 Gonka 以太坊桥中，Safe 多签保护着一个有限的以太坊侧管理员权限。该管理员权限仅用于极端恢复场景，例如在 30 天超时期间长时间未收到有效的 BLS 群组密钥更新，或出现 BLS 密钥冲突。在此类情况下，管理员可能能够设置或重置 BLS 群组密钥——这一权限具有较高的安全敏感性，因为 BLS 密钥最终控制着桥接签名。多签机制确保这些罕见的恢复操作不能由单个人完成，必须达到指定数量的签名者共同审查并批准交易后方可执行。

## 1. 前置要求

在您被添加到多签之前，请确保以下事项已准备就绪：

*   **以太坊钱包地址**：一个您可控的安全以太坊地址。
*   **硬件钱包（强烈推荐）**：出于主网安全考虑，强烈建议使用硬件钱包（如 Ledger 或 Trezor），而非仅依赖软件“热”钱包。
*   **网络访问能力**：确保您可以访问 [app.safe.global](https://app.safe.global)，并在您的钱包中配置了 RPC 提供商（如 Infura、Alchemy 或公共节点）。
*   **少量 ETH**：虽然签名交易通常是免费的（链下签名），但最终**执行**交易的人需要支付 ETH 作为燃气费（gas）。建议每位签名者至少持有 0.05
 0.1 ETH，以应对紧急执行需求。

## 2. 加入流程

### 第一步：提供您的地址
将您的公开以太坊地址（0x...）发送给多签管理员。**请仔细核对每一个字符**。切勿发送您的私钥或助记词。

### 第二步：添加所有者的交易
当前多签所有者将发起一笔“添加所有者”（Add Owner）的交易。该交易需根据当前签名阈值（例如 2/3）进行签名和执行。

### 第三步：访问 Safe
当交易在区块链上确认后：

1.  访问 [app.safe.global](https://app.safe.global)。
2.  连接您的钱包。
3.  添加 Safe。
4.  输入管理员提供的多签地址。
5.  您现在应在 Safe 仪表板中看到该 Safe，并且您的地址已列在“所有者”（Owners）列表中。

## 3. 如何签名交易

参与者最常见的任务是审查并签署已提议的交易。

1.  **审查**：
 
 
    *   打开 Safe 仪表板。
 
 
    *   进入 **“交易”（Transactions）** 标签页 → **“队列”（Queue）**。
 
 
    *   点击交易以展开详细信息。
2.  **签名**：点击 **“签名”（Sign）** 按钮。您的钱包将弹出请求签名提示。这通常是链下签名，不产生燃气费用。
3.  **等待**：当达到所需签名数量后，交易即可被执行。

## 4. 如何执行交易

如果您是最后一个签名者，或被指派负责完成交易：

1.  当所需数量的签名（例如 2 或 3 个）收集完成后，**“执行”（Execute）** 按钮将变为可用状态。
2.  点击 **“执行”**。
3.  您的钱包将提示您支付燃气费以将交易提交至区块链。
4.  一旦交易被矿工打包上链，操作即告完成。

## 5. 安全最佳实践

*   **验证 Safe 地址**：始终确保您正在与正确的 Safe 地址交互。首次加载后建议收藏该链接。
*   **检查交易详情**：恶意行为者可能试图发起将所有权转移给自己的交易。切勿签署您不理解其目标地址或用途的交易。
*   **做好备份**：确保您的硬件钱包助记词已安全离线备份。如果您丢失密钥访问权限，其他签名者需通过多数投票才能恢复您的访问权，多签本身无法自动重置。
*   **使用官方应用**：仅使用 [app.safe.global](https://app.safe.global)。警惕钓鱼网站。
