### 从以太坊封装ERC-20代币（例如USDT）到Gonka
1. **存款**：ERC-20代币所有者将其代币发送到以太坊上的桥接智能合约地址。
2. **锁定与铸造**：代币被锁定在合约中。每个Gonka节点运行一个小型桥接容器来监控桥接地址。一旦存款在以太坊上**最终确认**，并且**超过50%的节点（按投票权计算）** 独立确认后，桥接将在Gonka链上作为CW-20代币铸造该ERC-20的封装版本。
3. **每个代币对应一个封装合约**：每个以太坊代币在Gonka上对应**一个**封装的CW-20合约（由链ID + 以太坊合约地址确定）。某种代币的首次存款会实例化该合约；后续对该**相同**代币的所有存款都将复用**同一**封装合约。只有桥接可以实例化这些合约或铸造其代币。
4. **所有权**：铸造后，封装代币的所有权将分配给从以太坊使用的相同私钥/公钥对派生的Gonka地址。此后，所有者可以自由地将封装代币转移到任何其他Gonka账户。逐步流程请参阅[存款USDT（以太坊 → Gonka）](deposit-usdt.md)。

!!! note 
    注册代币（参见[注册桥接代币](register-token.md)）是可选的。它不影响代币是否可以桥接——仅附加元数据，使封装代币在钱包和仪表板中显示正确的名称/符号/小数位，并使其有资格加入链上流动性池。USDT和USDC已预注册。您可以在不先注册的情况下桥接和测试任何其他ERC-20代币。

### 解封/提现回以太坊
1. **请求**：所有者在Gonka链上提交特殊的提现交易。这将锁定/销毁封装代币并触发BLS签名生成。
2. **签名获取**：使用提供的API端点检查签名生成状态。
3. **执行**：一旦生成BLS签名，便用于向以太坊上的桥接合约发送提现指令。合约验证签名后，将原始代币释放到目标以太坊地址。逐步流程请参阅[提现USDT（Gonka → 以太坊）](withdraw-usdt.md)。

### 将原生GNK封装到以太坊（WGNK）
1. **托管**：一笔特殊交易将GNK锁定在托管账户中，并触发BLS签名生成。
2. **执行**：生成的BLS签名被提交到以太坊上的桥接合约，以在目标以太坊地址铸造等额的WGNK。逐步流程请参阅[存款GNK（Gonka → 以太坊）](deposit-gnk.md)。

!!! note 
    GNK永远不会以“原生”形式存在于以太坊上。在以太坊一侧，它始终是封装的**WGNK** ERC-20代币——而该代币**就是桥接合约本身**（相同地址；桥接合约也是WGNK ERC-20）。"将GNK桥接到以太坊"意味着在Gonka上将原生GNK锁定在托管中，并在以太坊上铸造等额的WGNK。

### 将WGNK从以太坊返回到GNK
1. **销毁**：WGNK被发送到以太坊上的桥接合约，合约将其销毁。
2. **释放**：一旦Gonka共识识别到销毁操作，等额的原生GNK将从托管中释放到从销毁WGNK的同一密钥派生的Gonka地址。详情请参阅[提现GNK（以太坊 → Gonka）](withdraw-gnk.md)。

### 将ETH作为WETH进行跨链
该跨链桥检测的是ERC-20代币转账，而非原生ETH转账。要将以太币转入Gonka：

1. **封装**：在以太坊上将你的ETH封装为**WETH**（标准的Wrapped Ether ERC-20代币）。
2. **跨链**：将WETH发送到跨链桥合约——其行为类似于[其他任何ERC-20代币的存入](deposit-usdt.md)。Gonka会将封装的WETH作为CW-20代币铸造到由同一私钥派生出的Gonka地址上，你可以通过相同方式将其提现回以太坊。

## Gonka → 以太坊的授权机制（每日组密钥）

每一笔从Gonka转出的交易（提现封装的ERC-20/ETH，或铸造WGNK）都需要由Gonka验证节点集通过BLS签名在以太坊上释放。为了让以太坊上的跨链桥合约信任该签名，它必须知道**当前纪元的组密钥**，而该密钥是通过每日签名链传递的：

* 每个纪元开始时（大约**每天一次**），Gonka会生成一个**新的组密钥**，并使用**上一个**纪元的密钥对其进行签名。
* 必须提交一笔小额交易到以太坊跨链桥合约以注册该新密钥。合约仅接受由前一个密钥签名的**下一个**顺序纪元密钥——因此密钥历史形成一条从创世延续至今的连续链条。
* **任何人**都可以使用相同的公开数据提交此更新。

一旦当前纪元的组密钥完成注册，提现就会变得快速：**单个纪元签名可以授权该纪元内发起的任意数量的提现**。用户的流程如下：

1. 在Gonka上提交提现/铸造操作，指定**接收的以太坊地址**。这会销毁/托管资产，并触发使用组密钥进行BLS签名。
2. 获取生成的签名。
3. 将签名连同转账数据提交至以太坊上的跨链桥合约，合约会使用当前组密钥验证签名，并向接收方释放ERC-20、ETH或WGNK。

### 如果当前纪元密钥尚未注册

提现使用当前纪元的组密钥签名，而跨链桥合约必须已持有该密钥。在纪元切换后（约每天一次），合约可能短暂滞后，你的释放操作可能会失败并显示`InvalidEpoch`，或仪表板可能显示跨链桥落后于链。你无需等待：任何人都可以通过仪表板或手动推送缺失的密钥更新。参见[跨链桥纪元更新](bridge-epoch-update.md)。

2. **释放**：一旦Gonka共识识别到销毁操作，等额的原生GNK将从托管中释放到从销毁WGNK的同一密钥派生的Gonka地址。请参阅[提现GNK（以太坊 → Gonka）](withdraw-gnk.md)。

### 桥接 ETH（作为 WETH）
桥接器检测 ERC-20 转账，而非原生 ETH 转账。要将以太币带入 Gonka：

1. **封装**：在以太坊上将您的 ETH 封装为 **WETH**（标准的 Wrapped Ether ERC-20）。
2. **桥接**：将 WETH 发送到桥接合约——其行为类似于 [其他任何 ERC-20 存款](deposit-usdt.md)。Gonka 会以与同一密钥派生的 Gonka 地址对应的 CW-20 代币形式铸造封装的 WETH，您也可以以相同方式将其提现回以太坊。

## Gonka → 以太坊授权机制（每日组密钥）

每次从 Gonka 转出（提现封装的 ERC-20/ETH 或铸造 WGNK）的操作，都由 Gonka 验证节点集的 BLS 签名在以太坊上释放。为了使以太坊桥接合约信任该签名，它必须知道 **当前纪元的组密钥**，而它通过每日签名链来获取该密钥：

* 每个纪元开始时（大约 **每天一次**），Gonka 会生成一个 **新的组密钥**，并使用 **前一个** 纪元的密钥对其进行签名。
* 必须向以太坊桥接合约提交一小笔交易以注册该新密钥。合约仅接受由前一个密钥签名的 **下一个** 顺序纪元密钥——因此密钥历史形成一条从创世延续至今的不间断链。
* **任何人** 都可以使用相同的公开数据提交此更新。

一旦当前纪元的组密钥完成注册，提现就会变得快速：**单个纪元签名可以授权该纪元期间发起的任意数量的提现**。用户的流程如下：

1. 在 Gonka 上提交提现/铸造操作，指定 **接收方以太坊地址**。这将销毁/托管资产，并触发使用组密钥进行的 BLS 签名。
2. 获取生成的签名。
3. 将签名连同转账数据提交到以太坊上的桥接合约，合约会使用当前组密钥验证签名，并向接收方释放 ERC-20、ETH 或 WGNK。
