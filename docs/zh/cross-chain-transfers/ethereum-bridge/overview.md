# 以太坊桥接概述

!!! warning 
    始终先进行小额测试交易。桥接转账是不可逆的，因此在转移大额资金之前，请先发送少量金额并确认其按预期到账。

    由Gonka共识控制的专用桥接智能合约已在以太坊上激活，地址为：

```text
0x972a7a92d92796a98801a8818bcf91f1648f2f68
```

[在Etherscan上查看桥接合约](https://etherscan.io/address/0x972a7a92d92796a98801a8818bcf91f1648f2f68){target=_blank}

!!! important "此地址同时也是WGNK代币"
    桥接合约**就是** **WGNK** ERC-20代币——它们是同一地址上的同一个合约，而不是两个独立的合约。因此上面的Etherscan页面同时代表桥接和WGNK代币。通过桥接转入Gonka的ERC-20代币在Gonka侧以CW-20代币形式存在；只有**WGNK**存在于以太坊上，即此合约。

!!! tip "将WGNK添加到您的以太坊钱包"
    WGNK是标准的ERC-20代币，因此您可以在任何支持ERC-20的钱包（如MetaMask、Trust Wallet、Ledger浏览器扩展等）中将其添加为**自定义代币**。使用上述桥接合约地址（`0x972a7a92d92796a98801a8818bcf91f1648f2f68`）作为代币合约地址——它既是桥接合约也是WGNK代币。

!!! note "安全审计"
    该桥接已由[CertiK](https://skynet.certik.com/projects/gonka){target=_blank}完成审计。完整报告可在CertiK Skynet页面获取（Gonka–以太坊桥接审计与共识及推理审计并列列出），也可查看本地副本：[CertiK Gonka–以太坊桥接审计报告（PDF）](../../assets/audits/CertiK-Gonka-Ethereum-Bridge-Audit.pdf){target=_blank}。

    桥接允许您转移：

* **任意ERC-20代币**（例如，[USDT](https://etherscan.io/token/0xdAC17F958D2ee523a2206206994597C13D831ec7){target=_blank}、[USDC](https://etherscan.io/token/0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48){target=_blank}、[WETH](https://etherscan.io/token/0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2){target=_blank}）从以太坊到Gonka，反之亦然。
* **原生Gonka币（GNK）** 到以太坊（作为封装的GNK）并返回。

!!! note "ETH以WETH形式桥接"
    桥接跟踪发送到桥接合约的**ERC-20代币转账**。原生ETH不是ERC-20代币，因此要桥接以太币，您必须先在以太坊上将其封装为**WETH**（标准的Wrapped Ether ERC-20代币），然后像其他任何ERC-20代币一样桥接WETH。

!!! important "任何ERC-20代币都可使用——无需注册"
    您可以桥接**任何**ERC-20代币，即使该代币从未在Gonka上注册过。当存款被识别后，桥接会自动创建封装的CW-20合约并铸造您的余额。通过治理进行注册是**可选的**，仅用于添加显示元数据（名称、符号、小数位）和交易资格——移动代币并不需要注册。请注意，未注册时，小数位可能与原始代币不匹配，这是正常现象。在代币注册前，其封装版本可能无法反映原始代币的小数位。这属于预期行为，不会影响您的余额。详情请参阅[注册桥接代币](register-token.md)。

!!! important "两个地址来自同一密钥"
    Gonka将封装代币发送到从签署以太坊存款的同一公钥派生的Gonka地址。如果您的以太坊和Gonka密钥来自同一个助记词，它们通常是**不同的**密钥，因此这将无法工作。不要使用助记词派生Gonka地址并假设它与您的以太坊地址匹配，两者派生方式不同。在首次转账前请阅读[地址与密钥](addresses-and-keys.md)。

## 概述

### 从以太坊封装ERC-20代币（例如USDT）到Gonka
1. **存款**：ERC-20代币所有者将其代币发送到以太坊上的桥接智能合约地址。
2. **锁定与铸造**：代币被锁定在合约中。每个Gonka节点运行一个小型桥接容器来监控桥接地址。一旦存款在以太坊上**最终确认**，并且**超过50%的节点（按投票权计算）** 独立确认后，桥接将在Gonka链上作为CW-20代币铸造该ERC-20的封装版本。
3. **每个代币对应一个封装合约**：每个以太坊代币在Gonka上对应**一个**封装的CW-20合约（由链ID + 以太坊合约地址确定）。某种代币的首次存款会实例化该合约；后续对该**相同**代币的所有存款都将复用**同一**封装合约。只有桥接可以实例化这些合约或铸造其代币。
4. **所有权**：铸造后，封装代币的所有权将分配给从以太坊使用的相同私钥/公钥对派生的Gonka地址。此后，所有者可以自由地将封装代币转移到任何其他Gonka账户。逐步流程请参阅[存款USDT（以太坊 → Gonka）](deposit-usdt.md)。

!!! note 
    注册代币（参见[注册桥接代币](register-token.md)）是可选的。它不影响代币是否可以桥接——仅附加元数据，使封装代币在钱包和仪表板中显示正确的名称/符号/小数位，并使其有资格加入链上流动性池。USDT和USDC已预注册。您可以在不先注册的情况下桥接和测试任何其他ERC-20代币。

### 解封/提现回以太坊
1. **请求**：所有者在Gonka链上提交特殊的提现交易。这将锁定/销毁封装代币并触发BLS签名生成。
2. **签名获取**：使用提供的API端点检查签名生成状态。
3. **执行**：一旦生成BLS签名，便用于向以太坊上的桥接合约发送提现指令。合约验证签名后，将原始代币释放到目标以太坊地址。逐步流程请参阅[提现USDT（Gonka → 以太坊）](withdraw-usdt.md)。

### 将原生GNK封装到以太坊（WGNK）
1. **托管**：一笔特殊交易将GNK锁定在托管账户中，并触发BLS签名生成。
2. **执行**：生成的BLS签名被提交到以太坊上的桥接合约，以在目标以太坊地址铸造等额的WGNK。逐步流程请参阅[存款GNK（Gonka → 以太坊）](deposit-gnk.md)。

!!! note 
    GNK永远不会以“原生”形式存在于以太坊上。在以太坊一侧，它始终是封装的**WGNK** ERC-20代币——而该代币**就是桥接合约本身**（相同地址；桥接合约也是WGNK ERC-20）。"将GNK桥接到以太坊"意味着在Gonka上将原生GNK锁定在托管中，并在以太坊上铸造等额的WGNK。

### 将WGNK从以太坊返回到GNK
1. **销毁**：WGNK被发送到以太坊上的桥接合约，合约将其销毁。
2. **释放**：一旦Gonka共识识别到销毁操作，等额的原生GNK将从托管中释放到从销毁WGNK的同一密钥派生的Gonka地址。详情请参阅[提现GNK（以太坊 → Gonka）](withdraw-gnk.md)。

### 将ETH作为WETH进行跨链
该跨链桥检测的是ERC-20代币转账，而非原生ETH转账。要将以太币转入Gonka：

1. **封装**：在以太坊上将你的ETH封装为**WETH**（标准的Wrapped Ether ERC-20代币）。
2. **跨链**：将WETH发送到跨链桥合约——其行为类似于[其他任何ERC-20代币的存入](deposit-usdt.md)。Gonka会将封装的WETH作为CW-20代币铸造到由同一私钥派生出的Gonka地址上，你可以通过相同方式将其提现回以太坊。

## Gonka → 以太坊的授权机制（每日组密钥）

每一笔从Gonka转出的交易（提现封装的ERC-20/ETH，或铸造WGNK）都需要由Gonka验证节点集通过BLS签名在以太坊上释放。为了让以太坊上的跨链桥合约信任该签名，它必须知道**当前纪元的组密钥**，而该密钥是通过每日签名链传递的：

* 每个纪元开始时（大约**每天一次**），Gonka会生成一个**新的组密钥**，并使用**上一个**纪元的密钥对其进行签名。
* 必须提交一笔小额交易到以太坊跨链桥合约以注册该新密钥。合约仅接受由前一个密钥签名的**下一个**顺序纪元密钥——因此密钥历史形成一条从创世延续至今的连续链条。
* **任何人**都可以使用相同的公开数据提交此更新。

一旦当前纪元的组密钥完成注册，提现就会变得快速：**单个纪元签名可以授权该纪元内发起的任意数量的提现**。用户的流程如下：

1. 在Gonka上提交提现/铸造操作，指定**接收的以太坊地址**。这会销毁/托管资产，并触发使用组密钥进行BLS签名。
2. 获取生成的签名。
3. 将签名连同转账数据提交至以太坊上的跨链桥合约，合约会使用当前组密钥验证签名，并向接收方释放ERC-20、ETH或WGNK。

### 如果当前纪元密钥尚未注册

提现使用当前纪元的组密钥签名，而跨链桥合约必须已持有该密钥。在纪元切换后（约每天一次），合约可能短暂滞后，你的释放操作可能会失败并显示`InvalidEpoch`，或仪表板可能显示跨链桥落后于链。你无需等待：任何人都可以通过仪表板或手动推送缺失的密钥更新。参见[跨链桥纪元更新](bridge-epoch-update.md)。

## 时间与最终确认

转账时间取决于方向：

* **以太坊 → Gonka：约15–20分钟**。跨链桥会在存款区块在以太坊上**最终确认**（≈两个纪元）后才进行铸造。Gonka不依赖任何预先垫资并承担风险的中间方，因此这一等待不可避免。确切时间还取决于你的交易落在以太坊纪元中的位置。
* **Gonka → 以太坊：快速**。组密钥每天生成一次并全天使用，因此你可以随时发起提现，仅需等待BLS签名和你在以太坊上的执行交易。

!!! warning "在Gonka链中断期间"
    跨链桥独立于Gonka区块生产，摄取**已最终确认的以太坊区块**。如果Gonka链停止运行，主机可以选择**跳过**这些以太坊区块（及其包含的存款）而不是排队，且在签名恢复前无法完成提现。如果你在[仪表板](tracker-integration.md)上看到链被报告为离线，请等待其恢复正常后再进行跨链操作。

## 跨链桥与交易所的区别

跨链桥仅**在一个链上锁定资产并在另一链上释放**——它不会将一种资产兑换为另一种。交易发生在**独立的**合约中：

* 在以太坊上，例如在Uniswap等DEX上交易**WGNK ↔ USDC**。
* 在Gonka上，通过链上流动性池交易封装代币。

因此典型的"在以太坊上卖出GNK"流程是：将GNK通过跨链桥转为WGNK至以太坊，然后在DEX上将WGNK交易为其他代币。

## 跨链桥与IBC的区别

此跨链桥直接连接Gonka与**以太坊**。Gonka也支持**IBC**用于与其他Cosmos链的转账（参见[IBC](../ibc/withdraw-usdt-via-kava.md)部分）。

* 使用**以太坊跨链桥**在**以太坊与Gonka之间**转移资产。通常这比通过IBC路由更简单，且需要的Gas代币更少。
* 从以太坊跨链而来的代币在Gonka上作为与此跨链桥绑定的封装CW-20代币存在。它可以在Gonka内部转移并**退回以太坊**，但**不能**被转发或出售到其他Cosmos链——如需此功能，应使用IBC原生资产。
