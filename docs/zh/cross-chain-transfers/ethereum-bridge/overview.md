# 以太坊跨链桥概述

!!! warning
    务必先进行小额测试交易。跨链转账是不可逆的，因此在转移大额资金之前，请先发送少量金额，并确认其如期到账。

由 Gonka 共识控制的专用跨链桥智能合约已在以太坊上激活，地址为：

```text
0x972a7a92d92796a98801a8818bcf91f1648f2f68
```

[在 Etherscan 上查看跨链桥合约](https://etherscan.io/address/0x972a7a92d92796a98801a8818bcf91f1648f2f68){target=_blank}

!!! important "该地址同时也是 WGNK 代币"
    跨链桥合约**就是** **WGNK** ERC-20 代币——它们是位于该地址的同一个合约，而非两个独立的合约。因此上面的 Etherscan 页面同时是跨链桥和 WGNK 代币。跨链桥**进入** Gonka 的包装 ERC-20 代币在 Gonka 侧以 CW-20 代币形式存在；在以太坊上只有 **WGNK**，而它就是该合约。

!!! note "安全审计"
    该跨链桥已由 [CertiK](https://skynet.certik.com/projects/gonka){target=_blank} 审计。完整报告可在 CertiK Skynet 页面获取（其中 Gonka – 以太坊跨链桥审计与共识和推理审计一同列出），同时也提供本地副本：[CertiK Gonka – 以太坊跨链桥审计 (PDF)](../../../assets/audits/CertiK-Gonka-Ethereum-Bridge-Audit.pdf){target=_blank}。

该跨链桥允许您进行以下资产的转移：

* **任何 ERC-20 代币**（例如 [USDT](https://etherscan.io/token/0xdAC17F958D2ee523a2206206994597C13D831ec7){target=_blank}、[USDC](https://etherscan.io/token/0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48){target=_blank}、[WETH](https://etherscan.io/token/0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2){target=_blank}）从以太坊转移到 Gonka，反之亦然。
* **Gonka 原生代币 (GNK)** 转移到以太坊（作为包装的 GNK），反之亦然。

!!! note "ETH 以 WETH 形式跨链"
    跨链桥追踪的是**发送至跨链桥合约的 ERC-20 代币转账**。原生 ETH 不是 ERC-20 代币，因此要跨链以太币，您需要先在以太坊上将其包装为 **WETH**（标准的 Wrapped Ether ERC-20 代币），然后像其他 ERC-20 代币一样跨链 WETH。

!!! important "任何 ERC-20 代币均可跨链——无需注册"
    您可以跨链**任何** ERC-20 代币，即使该代币从未在 Gonka 上注册过。当充值被识别后，跨链桥会自动创建包装的 CW-20 合约并铸造您的余额。通过治理进行注册是**可选的**，仅用于添加显示元数据（名称、符号、精度）以及交易资格——跨链代币**不需要**注册。请注意，未注册的代币精度可能与原始代币不一致，这是预期行为。在代币注册之前，其包装版本可能不会反映原始代币的精度。详见[注册跨链桥代币](register-token.md)。

!!! important "两个地址源自同一密钥"
    Gonka 将包装代币发送到由**签署以太坊充值交易的同一公钥派生的** Gonka 地址。如果您的以太坊和 Gonka 密钥来自助记词，它们通常是**不同的**密钥，这种情况下跨链将不会按预期工作。不要使用助记词派生 Gonka 地址并假设它与您的以太坊地址匹配——两者的派生方式不同。请在首次转账前阅读[地址与密钥](addresses-and-keys.md)。

## 概述

### 将 ERC-20 代币（如 USDT）从以太坊包装转移至 Gonka
1. **充值 (Deposit)**：ERC-20 代币的所有者将代币发送到以太坊上的跨链桥智能合约地址。
2. **锁定与铸造 (Locking & Minting)**：代币在合约中被锁定。每个 Gonka 主机运行一个小型跨链桥容器来监控跨链桥地址。一旦充值在以太坊上被**最终确认**，且**超过 50% 的主机（按投票权计算）**独立确认了该笔充值，跨链桥即在 Gonka 链上铸造该 ERC-20 代币的包装版本（作为 CW-20 代币）。
3. **每个代币一个包装合约**：每个以太坊代币精确映射到 Gonka 上的**一个**包装 CW-20 合约（以链 ID + 以太坊合约地址为键）。首次充值某个代币时会创建该合约；之后对**同一**代币的每次充值都会复用**同一个**包装合约。只有跨链桥才能创建这些合约或铸造其代币。
4. **所有权分配 (Ownership)**：铸造完成后，包装代币的所有权将被分配给由所有者在以太坊上使用的同一私钥/公钥对派生出的 Gonka 地址。自此，所有者可以自由地将包装代币转账给任何其他 Gonka 账户。详见[存入 USDT（以太坊 → Gonka）](deposit-usdt.md)了解分步流程。

!!! note
    注册代币（参见[注册跨链桥代币](register-token.md)）是可选的。它不影响代币是否可以跨链——它只是附加元数据，使包装代币在钱包和仪表板中显示正确的名称/符号/精度，并使其有资格参与链上流动性池。USDT 和 USDC 已预先注册。您可以在不注册的情况下跨链和测试任何其他 ERC-20 代币。

### 解包装 / 提取回以太坊
1. **请求 (Request)**：所有者在 Gonka 链上提交一个特殊的提取交易。这将锁定/销毁包装代币，并触发 BLS 签名生成。
2. **签名检索 (Signature Retrieval)**：使用提供的 API 端点检查签名生成的状态。
3. **执行 (Execution)**：BLS 签名生成后，将其用于向以太坊上的跨链桥合约发送提取命令。合约验证签名并将原始代币释放到目标以太坊地址。详见[提取 USDT（Gonka → 以太坊）](withdraw-usdt.md)了解分步流程。

### 将原生 GNK 包装转移至以太坊 (WGNK)
1. **托管 (Escrow)**：通过一笔特殊交易将 GNK 锁定在托管账户中，并触发 BLS 签名生成。
2. **执行 (Execution)**：将生成的 BLS 签名提交至以太坊上的跨链桥合约，从而向目标以太坊地址铸造 WGNK。详见[存入 GNK（Gonka → 以太坊）](deposit-gnk.md)了解分步流程。

!!! note
    GNK 在以太坊上从不以"原生"形式存在。在以太坊侧，它始终是包装后的 **WGNK** ERC-20 代币——而该代币**就是跨链桥合约本身**（同一地址；跨链桥合约同时也是 WGNK ERC-20）。"将 GNK 跨链至以太坊"意味着在 Gonka 上将原生 GNK 锁定在托管账户中，并在以太坊上铸造等量的 WGNK。

### 将 WGNK 从以太坊返回为 GNK
1. **销毁 (Burn)**：将 WGNK 发送至以太坊上的跨链桥合约，合约将其销毁。
2. **释放 (Release)**：一旦销毁被 Gonka 共识识别，等量的原生 GNK 将从托管账户释放到由销毁 WGNK 所用同一密钥派生的 Gonka 地址。详见[提取 GNK（以太坊 → Gonka）](withdraw-gnk.md)。

### 跨链 ETH（作为 WETH）
跨链桥检测的是 ERC-20 转账，而非原生 ETH 转账。要将以太币带入 Gonka：

1. **包装 (Wrap)**：在以太坊上将您的 ETH 包装为 **WETH**（标准的 Wrapped Ether ERC-20）。
2. **跨链 (Bridge)**：将 WETH 发送至跨链桥合约——它的行为与[任何其他 ERC-20 充值](deposit-usdt.md)相同。Gonka 会将包装的 WETH 作为 CW-20 代币铸造到由同一密钥派生的 Gonka 地址，您可以用相同的方式将其提取回以太坊。

## Gonka → 以太坊的授权方式（每日群组密钥）

每笔从 Gonka **转出**的交易（提取包装的 ERC-20/ETH，或铸造 WGNK）都需要 Gonka 验证者集的 BLS 签名才能在以太坊上释放。为了让以太坊跨链桥合约信任该签名，它必须知道**当前纪元的群组密钥**，而这通过每日的签名链来实现：

* 每个纪元开始时（大约**每天一次**），Gonka 生成一个**新的群组密钥**，并用**上一个**纪元的密钥对其签名。
* 需要向以太坊跨链桥合约提交一笔小额交易来注册该新密钥。合约只接受**下一个**顺序的纪元密钥，且该密钥必须由上一个纪元密钥签名——因此密钥历史形成一条从创世开始的不间断链条。
* **任何人**都可以使用相同的公开数据提交此更新。

一旦当前纪元的群组密钥注册完成，提现就很快：**一个纪元签名可以授权该纪元内发起的任意数量的提现**。用户的流程如下：

1. 在 Gonka 上提交提现/铸造操作，指定**接收方以太坊地址**。这会销毁/托管资产并触发使用群组密钥的 BLS 签名。
2. 检索生成的签名。
3. 将签名和转账数据提交至以太坊上的跨链桥合约，合约根据当前群组密钥进行验证，并向接收方释放 ERC-20、ETH 或 WGNK。

### 如果当前纪元密钥尚未注册

提现使用当前纪元的群组密钥签名，跨链桥合约必须已经持有该密钥。在纪元刚切换后（大约每天一次），合约可能会短暂滞后，您的释放交易可能因 `InvalidEpoch` 而失败，或仪表板可能显示跨链桥落后于链。您无需等待：任何人都可以推送缺失的密钥更新，可以通过仪表板或手动操作。详见[跨链桥纪元更新](bridge-epoch-update.md)。

## 时间与最终确认

转账时间取决于方向：

* **以太坊 → Gonka：约 15–20 分钟。**跨链桥等待充值区块在以太坊上被**最终确认**（约两个 epoch），然后才铸造。Gonka 不使用预付资金和承担风险的中间方，因此这个等待是不可避免的。确切时间还取决于您的交易落在以太坊 epoch 中的位置。
* **Gonka → 以太坊：很快。**群组密钥每个纪元生成一次并全天使用，因此您可以随时开始提现，只需等待 BLS 签名和您的以太坊执行交易。

!!! warning "Gonka 链中断期间"
    跨链桥独立于 Gonka 出块来接收**已最终确认的以太坊区块**。如果 Gonka 链停止运行，这些以太坊区块（及其中的充值）可能会被主机**跳过**而不是排队，提现也无法签名直到签名恢复。如果您在[仪表板](tracker-integration.md)上看到链被报告为停机，请等到它恢复正常后再进行跨链。

## 跨链桥 vs. 交易所

跨链桥仅**在一条链上锁定资产并在另一条链上释放**——它不会将一种资产兑换为另一种。交易发生在**独立的**合约上：

* 在以太坊上，例如在 Uniswap 等 DEX 上兑换 **WGNK ↔ USDC**。
* 在 Gonka 上，通过链上流动性池兑换包装代币。

因此典型的"在以太坊上出售 GNK"流程是：将 GNK → WGNK 跨链到以太坊，然后在 DEX 上将 WGNK 兑换为另一种代币。

## 跨链桥 vs. IBC

此跨链桥将 Gonka 直接连接到**以太坊**。Gonka 还支持 **IBC** 与其他 Cosmos 链进行转账（参见 [IBC](../ibc/withdraw-usdt-via-kava.md) 部分）。

* 使用**以太坊跨链桥**在**以太坊和 Gonka** 之间转移资产。它通常比通过 IBC 路由更简单，所需的 Gas 代币也更少。
* 从以太坊跨链的代币在 Gonka 上作为与此跨链桥绑定的包装 CW-20 存在。它可以在 Gonka 内部转账并**返回以太坊**，但**不能**被转发或卖到另一条 Cosmos 链——为此您需要使用 IBC 原生资产。
