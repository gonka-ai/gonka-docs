# 以太坊桥概述

!!! warning
    务必先进行小额测试交易。桥接转账是不可逆的，因此在转移大额资金前，请先发送少量金额并确认其如期到账。

    由 Gonka 共识控制的专用桥接智能合约已在以太坊上激活，合约地址为：

```text
0x972a7a92d92796a98801a8818bcf91f1648f2f68
```

[在 Etherscan 上查看桥接合约](https://etherscan.io/address/0x972a7a92d92796a98801a8818bcf91f1648f2f68){target=_blank}

!!! important "此地址同时也是 WGNK 代币"
    桥接合约 **就是** **WGNK** ERC-20 代币 —— 它们是同一个合约，位于同一地址，并非两个独立合约。因此上述 Etherscan 页面同时代表了桥接合约和 WGNK 代币。通过桥接转入 Gonka 链的 ERC-20 代币在 Gonka 侧以 CW-20 代币形式存在；只有 **WGNK** 存在于以太坊上，而它正是这个合约。

!!! note "安全审计"
    该桥接合约已由 [CertiK](https://skynet.certik.com/projects/gonka){target=_blank} 完成审计。完整报告可在 CertiK Skynet 页面查看（其中列出了 Gonka–以太坊桥接审计，以及共识与推理审计），也可查看本地副本：[CertiK Gonka–以太坊桥接审计报告（PDF）](../../assets/audits/CertiK-Gonka-Ethereum-Bridge-Audit.pdf){target=_blank}。

    该桥接允许你转移：

* **任意 ERC-20 代币**（例如 [USDT](https://etherscan.io/token/0xdAC17F958D2ee523a2206206994597C13D831ec7){target=_blank}、[USDC](https://etherscan.io/token/0xA0b86991c6218b36c1d19D4a2e9Eb0cE3606eB48){target=_blank}、[WETH](https://etherscan.io/token/0xC02aaA39b223FE8D0A0e5C4F27eAD9083C756Cc2){target=_blank}）从以太坊到 Gonka，或反向转移。
* **原生 Gonka 币（GNK）** 到以太坊（作为封装的 GNK）并返回。

!!! note "ETH 以 WETH 形式进行桥接"
    桥接合约追踪的是发送至其地址的 **ERC-20 代币转账**。原生 ETH 并非 ERC-20 代币，因此若要桥接以太币，你必须先在以太坊上将其封装为 **WETH**（标准的 Wrapped Ether ERC-20 代币），然后像其他任何 ERC-20 代币一样桥接 WETH。

!!! important "任意 ERC-20 代币均可使用 —— 无需注册"
    你可以桥接 **任何** ERC-20 代币，即使该代币从未在 Gonka 上注册过。当存款被识别后，桥接合约会自动创建对应的封装 CW-20 合约并铸造你的余额。通过治理进行注册是 **可选的**，仅用于添加显示元数据（名称、符号、小数位数）和交易资格 —— **并非** 转移代币所必需。请注意，未注册时，小数位数可能与原始代币不一致，这是正常现象。在代币注册前，其封装版本可能无法反映原始代币的小数位数。这不会影响你的余额。详情请参阅 [注册桥接代币](register-token.md)。

!!! important "两个地址来自同一私钥"
    Gonka 将封装代币发送至 **由签署以太坊存款交易的公钥推导出的 Gonka 地址**。如果你的以太坊和 Gonka 密钥均来自助记词，它们通常是 **不同的密钥**，因此无法匹配。不要假设使用同一助记词生成的 Gonka 地址会与你的以太坊地址对应，因为两者的推导方式不同。请在首次转账前阅读 [地址与密钥](addresses-and-keys.md)。

## 概述

### 将 ERC-20 代币（如 USDT）从以太坊桥接到 Gonka
1. **存款**：ERC-20 代币所有者将代币发送至以太坊上的桥接智能合约地址。
2. **锁定与铸造**：代币被锁定在合约中。每个 Gonka 节点运行一个小型桥接容器，监控桥接地址。一旦存款在以太坊上 **最终确认**，且 **超过 50% 的节点（按投票权计算）** 独立确认后，桥接将在 Gonka 链上铸造该 ERC-20 代币的封装版本，形式为 CW-20 代币。
3. **每种代币对应一个封装合约**：每个以太坊代币在 Gonka 上对应 **唯一一个** 封装 CW-20 合约（通过链 ID
4. 以太坊合约地址确定）。某种代币的首次存款将创建该合约；后续对该 **相同** 代币的存款将复用 **同一** 封装合约。只有桥接合约能创建这些合约或铸造其代币。
**所有权**：铸造完成后，封装代币的所有权归属于由以太坊交易所用私钥/公钥对推导出的 Gonka 地址。此后，所有者可自由将封装代币转账至任意其他 Gonka 账户。参见 [存入 USDT（以太坊 → Gonka）](deposit-usdt.md) 了解详细步骤。

!!! note
    注册代币（参见 [注册桥接代币](register-token.md)）是可选的。它不影响代币是否可桥接 —— 仅附加元数据，使封装代币在钱包和仪表板中显示正确的名称/符号/小数位，并使其有资格加入链上流动性池。USDT 和 USDC 已预注册。你可以在不注册的情况下桥接和测试任何其他 ERC-20 代币。

### 解封装 / 从 Gonka 提现回以太坊
1. **请求**：所有者在 Gonka 链上提交特殊的提现交易。该操作将锁定/销毁封装代币，并触发 BLS 签名生成。
2. **获取签名**：使用提供的 API 端点检查签名生成状态。
3. **执行**：一旦生成 BLS 签名，即可将其用于向以太坊上的桥接合约发送提现指令。合约验证签名后，将原始代币释放至目标以太坊地址。参见 [提现 USDT（Gonka → 以太坊）](withdraw-usdt.md) 了解详细步骤。

### 将原生 GNK 封装为以太坊上的 WGNK
1. **托管**：通过特殊交易将 GNK 锁定在托管账户中，并触发 BLS 签名生成。
2. **执行**：将生成的 BLS 签名提交至以太坊上的桥接合约，以在目标以太坊地址上铸造 WGNK。参见 [存入 GNK（Gonka → 以太坊）](deposit-gnk.md) 了解详细步骤。

!!! note
    GNK 永远不会以“原生”形式存在于以太坊上。在以太坊一侧，它始终是封装的 **WGNK** ERC-20 代币 —— 而该代币 **就是桥接合约本身**（同一地址；桥接合约同时也是 WGNK ERC-20 合约）。"将 GNK 桥接到以太坊" 指的是在 Gonka 上将原生 GNK 锁定在托管账户中，并在以太坊上铸造等量的 WGNK。

### 将 WGNK 从以太坊返回至 GNK
1. **销毁**：将 WGNK 发送至以太坊上的桥接合约，合约将其销毁。
2. **释放**：一旦 Gonka 共识识别到该销毁操作，等量的原生 GNK 将从托管账户释放至由销毁 WGNK 所用密钥推导出的 Gonka 地址。参见 [提现 GNK（以太坊 → Gonka）](withdraw-gnk.md)。

### 桥接 ETH（作为 WETH）
桥接合约检测的是 ERC-20 转账，而非原生 ETH 转账。要将以太币带入 Gonka：

1. **封装**：在以太坊上将你的 ETH 封装为 **WETH**（标准的 Wrapped Ether ERC-20 代币）。
2. **桥接**：将 WETH 发送至桥接合约 —— 其行为与 [其他任何 ERC-20 存款](deposit-usdt.md) 相同。Gonka 将在由同一密钥推导出的 Gonka 地址上铸造封装的 WETH 作为 CW-20 代币，你也可以以相同方式将其提现回以太坊。

## Gonka → 以太坊转账的授权机制（每日组密钥）

每次从 Gonka **转出**（提现封装的 ERC-20/ETH，或铸造 WGNK）的操作，都由 Gonka 验证节点集合生成的 BLS 签名在以太坊上释放。为了让以太坊桥接合约信任该签名，它必须知道 **当前纪元的组密钥**，而该密钥通过每日签名链传递：

* 每个纪元开始时（约 **每天一次**），Gonka 会生成一个 **新的组密钥**，并使用 **上一个** 纪元的密钥对其进行签名。
* 必须向以太坊桥接合约提交一笔小型交易来注册该新密钥。合约仅接受由前一个密钥签名的 **下一个** 顺序纪元密钥 —— 因此密钥历史形成一条从创世延续至今的不可中断链。
* **任何人** 都可以使用相同的公开数据提交此更新。

一旦当前纪元的组密钥注册成功，提现将变得快速：**单个纪元签名可授权该纪元内发起的任意数量的提现**。用户流程如下：

1. 在 Gonka 上提交提现/铸造操作，指定 **接收方以太坊地址**。该操作将销毁/托管资产，并触发使用组密钥的 BLS 签名。
2. 获取生成的签名。
3. 将签名及转账数据提交至以太坊上的桥接合约，合约使用当前组密钥验证签名，并将 ERC-20、ETH 或 WGNK 释放至接收方。

### 如果当前纪元密钥尚未注册

提现使用当前纪元的组密钥签名，而桥接合约必须已持有该密钥。在纪元切换后（约每天一次），合约可能短暂滞后，你的释放操作可能失败并提示 `InvalidEpoch`，或仪表板显示桥接落后于链。你无需等待：任何人可通过仪表板或手动推送缺失的密钥更新。参见 [桥接纪元更新](bridge-epoch-update.md)。

## 时间与最终确认

转账时间取决于方向：

* **以太坊 → Gonka：约 15–20 分钟**。桥接会在存款区块在以太坊上 **最终确认**（约两个纪元）后才进行铸造。Gonka 不依赖任何代垫资金并承担风险的中间方，因此此等待不可避免。确切时间还取决于你的交易落在以太坊纪元中的位置。
* **Gonka → 以太坊：快速**。组密钥每天生成一次并全天使用，因此你可随时发起提现，仅需等待 BLS 签名和以太坊执行交易。

!!! warning "在 Gonka 链中断期间"
    桥接独立于 Gonka 区块生产，直接摄取 **已最终确认的以太坊区块**。如果 Gonka 链暂停，这些以太坊区块（及其包含的存款）可能被节点 **跳过** 而非排队，且在签名恢复前无法执行提现。如果你在 [仪表板](tracker-integration.md) 上看到链被报告为宕机，请等待链恢复正常后再进行桥接。

## 桥接与交易所的区别

桥接仅 **在一个链上锁定资产，并在另一链上释放等量资产** —— 它不进行资产互换。交易发生在 **独立的** 合约中：

* 在以太坊上，例如在 Uniswap 等 DEX 上交易 **WGNK ↔ USDC**。
* 在 Gonka 上，通过链上流动性池交易封装代币。

因此，典型的“在以太坊出售 GNK”流程是：先将 GNK 桥接到以太坊成为 WGNK，然后在 DEX 上将 WGNK 兑换为其他代币。

## 桥接与 IBC 的区别

此桥接直接连接 Gonka 与 **以太坊**。Gonka 还支持 **IBC** 用于与其他 Cosmos 链的资产转移（参见 [IBC](../ibc/withdraw-usdt-via-kava.md) 章节）。

* 使用 **以太坊桥接** 在 **以太坊与 Gonka** 之间转移资产。通常比通过 IBC 路由更简单，且需要的 gas 代币更少。
* 从以太坊桥接的代币在 Gonka 上作为与该桥接绑定的封装 CW-20 代币存在。它可在 Gonka 内部转移并 **返回以太坊**，但 **不能** 被转发或出售至其他 Cosmos 链 —— 若要实现这一点，应使用 IBC 原生资产。
