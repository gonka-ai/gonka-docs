# 术语表

**AI Token** 是一种与模型相关的计算单位，用于量化训练或推理操作所需的计算成本。例如，在 QwQ-32B 模型（FP16，32K 上下文）的场景中，一个 AI Token 可能代表处理固定数量输入和/或输出 Token 所需的资源。AI Token 与基础模型的特性紧密相关，反映了实际的内存、FLOP 等消耗。

**抵押品（Collateral）** 机制允许参与者锁定 GNK 代币，以激活其已获得的“计算证明”（Proof of Compute, PoC）权重的一部分。投票权从不单纯来源于持有代币。GNK 代币作为经济抵押品，而非影响力的来源。影响力通过持续的计算贡献获得，而锁定 GNK 抵押品则是参与治理并确保责任履行的前提。

**确认/随机计算证明（cPoC）** 是一种辅助验证机制，用于在主 [计算证明](https://gonka.ai/glossary/#glossary:~:text=Proof%20of%20Compute%20(PoC)%20is,is%20the%20execution%20phase%20of%20PoC.)（PoC）冲刺阶段之外，验证节点声明的计算权重的稳定性。这些检查旨在确认节点是否持续提供与其最近 PoC 结果相匹配的计算能力。CPoC 不替代标准的 PoC 流程（即 Sprint）。

**确认比率（Confirmation Ratio）** 是指节点在 PoC 阶段获得的周期权重与其在 cPoC 中获得的权重之间的比值。较高的确认比率表明该节点提供的计算资源更稳定。如果确认比率低于或等于 50%，则该节点将被取消当期奖励资格。

**共识（Consensus）** 是网络就区块链单一可验证状态达成一致的协议，确保所有参与者维护一致且防篡改的账本。在 Gonka 网络中，共识通过 PoC 实现。

**周期（Epoch）** 是链的核心运行周期。在一个周期内，节点执行有意义的 AI 任务，参与验证，并累积奖励。一个周期持续 17280 个区块（约 24 小时），并在 Sprint 完成后结束，确定下一周期的节点权重。

**Gonka 区块链** 是去中心化 AI 网络的基础账本和协调层（L1）。它记录余额、交易以及证明节点正确执行 AI 任务的加密凭证，而所有实际计算（如推理和训练）均在链下进行。

**Gonka 网络** 是一个由参与者（包括节点和开发者）组成的综合生态系统，通过去中心化基础设施进行交互。依托 Gonka 区块链，该网络分发任务、验证结果，并仅对可验证的有用工作给予奖励，从而为 AI 工作负载创造一个具有竞争力且可扩展的环境。

**Gonka 角色：**

- **开发者（Developer）**：通过利用网络的分布式算力来构建和部署 AI 应用。
- **Gonka 贡献者（Gonka Contributor）**：参与核心区块链代码库的开发、协议升级、性能优化、安全补丁和新功能集成。
- **持有者（Holder）**：持有网络的原生代币，即拥有一个包含 GNK 代币的 Gonka 钱包。持有者可以持有、转账或出售代币，也可将其用于支付推理费用，并根据协议规则使用。作为持有者并不意味着承担任何义务、责任或治理角色，仅限于标准的代币所有权。
- **节点（Host）**：向网络贡献计算能力。节点执行推理和其他计算任务，并根据其贡献的计算能力按比例获得奖励，前提是保持诚实参与和可靠性。节点是网络的支柱。只有节点拥有网络中的投票权。这种投票权代表其在治理中的权重，用于提出和投票决定协议决策、参数变更和升级。任何节点都同时担任验证者、传输代理和执行者（这些不是预定义或链上角色，而是处理推理请求时动态承担的操作职能）。

**ML 节点（ML Node）** 在 Gonka 中是节点私有基础设施内的一个计算节点，负责执行所有真实的 AI 工作负载（如大语言模型推理、训练步骤和 Sprint 计算）。它从不直接与公共网络交互，而是从节点的 API 节点接收任务，并将结果返回给 API 节点。ML 节点在链下执行所有繁重的计算，而区块链仅记录验证凭证、证明和任务承诺。

**网络节点（Network Node）** 是处理所有通信的服务，包括：

- [**链节点（Chain Node）**](https://github.com/product-science/inference-ignite/tree/main/inference-chain)：连接区块链，维护区块链层，并处理共识。
- [**API 节点（API Node）**](https://github.com/product-science/inference-ignite/tree/main/decentralized-api)：作为区块链（链节点）与 AI 执行环境（ML 节点）之间的主要协调层。它提供 REST/gRPC 接口，供用户、开发者和内部组件交互，同时管理任务编排、验证调度和需要链下执行的结果验证流程。除了处理用户请求外，它还负责：
 
 
    - 将推理和训练任务路由到 ML 节点
 
 
    - 记录推理结果并确保任务完成
 
 
    - 调度和管理验证任务
 
 
    - 向链节点报告收据和签名以参与共识
 
 
    - 协调 PoC 流程

**私钥（Private Key）** 是密码学密钥对中的私有部分，对保障和授权 Gonka 网络内的交易和操作至关重要。私钥必须始终保密。任何获得私钥的一方都可以在协议层面代表其所有者行事。保护私钥完全由持有者自行负责。

**计算证明（Proof of Compute, PoC）** 是一种共识机制，用可验证的基于 Transformer 的计算能力取代基于资本或哈希算力的权重机制。它定义了如何衡量真实 AI 计算并将其转化为治理和共识权重。PoC 通过每个周期末尾进行的短时间同步“冲刺”（Sprint）来执行。在冲刺之外的时间，周期用于执行现实世界的 AI 计算。实际上，“计算证明”（PoC）和“冲刺”（Sprint）这两个术语经常互换使用。当提到“下一次 PoC”或“PoC 阶段”时，通常指的是下一次 Sprint，即 PoC 的执行阶段。

**公钥（Public Key）** 是密码学密钥对中可公开的部分，用于验证签名、加密消息以及在 Gonka 网络中标识账户。这是密钥对中可公开分享的部分。

**随机任务验证（Randomized Task Verification）** 是平台验证策略的基础。系统不会对每个推理任务都进行冗余验证，而是根据节点（执行者）的声誉随机选择一部分任务进行验证。节点声誉越高，其工作需要被验证的比例越低。这种方法将验证开销大幅降低至仅 1–10% 的任务量，同时通过概率性保证和作弊被发现后将失去奖励的威慑机制维持信任。

**冲刺（Sprint）** 是 PoC 的一个阶段。在此期间，所有节点同时在一个具有随机化层的 Transformer 模型上运行与 AI 相关的推理任务，处理一串随机数（nonce）并生成输出向量。节点在下一周期的投票权与其成功处理的 nonce 数量成正比，前提是报告的输出能够被验证为由指定的 Sprint 模型生成。

**冲刺种子（Sprint Seed）** 由基于最新区块链状态的随机数生成器生成。该种子用于对 Transformer 模型的隐藏层应用随机变换。

**验证者 `status: jailed`** 表示该验证者因未能满足最低共识参与要求（具体而言，在规定窗口期内签署的区块少于所需数量），已被协议自动且临时地从出块流程中移除。

**投票权（权重，Voting Power / Weight）** 代表节点在网络治理和协议决策中的权重，由其在 Sprint 中处理的 nonce 数量按比例决定。
