# 密钥管理架构

本文档描述了 Gonka Network 的全面密钥管理架构，这是一个去中心化的人工智能基础设施，需要通过基于角色的密钥分离来实现强大的安全性。

## 概述

Gonka Network 实施了一种**基于角色的密钥管理系统**，将自动化功能与高风险的手动审批分离开来。该架构确保没有单一密钥可以控制所有网络操作，从而提供更高的安全性和操作灵活性。

!!! info "快速设置"
    如需立即部署，请参阅 [快速入门指南](/host/quickstart/)。本文档重点介绍完整的密钥管理架构和安全模型。

## 安全模型

### 账户密钥（冷钱包）
 
- **关键**
- 主控密钥，用于授权所有其他密钥
- 必须离线存储在安全、空气隔离的设备上
- 仅用于权限授予和验证节点注册
- 由助记词保护 —— **一旦丢失，所有访问权限将永久丧失**

### 机器学习操作密钥（温钱包）
- 由账户密钥授权，专用于机器学习相关交易
- 以加密文件形式存储于服务器，可编程访问
- 用于自动化交易（推理请求、证明提交、奖励领取等）
- 可随时由账户密钥进行轮换或撤销

### 共识密钥（TMKMS
 
- 温存储）
- 由安全的 TMKMS 服务管理
- 温存储模式，具备双重签名防护机制
- 用于区块验证和参与网络共识
- 可通过账户密钥或授权代理进行轮换

## 最佳实践

### 安全准则

1. **账户密钥保护**
    - 使用加密存储并尽量减少联网的安全本地设备
 
 
      - *安全本地设备*：专用计算机，访问受限，不用于日常浏览/邮件，理想情况下应为空气隔离或网络连接受限
 
 
  - 使用 `file` 或 `os` 密钥环后端进行安全本地存储
 
 
  - 为密钥环设置强且唯一的密码
 
 
  - 在安全位置离线备份助记词
 
 
  - 切勿用于常规操作 —— 仅限权限授予和验证节点操作

2. **硬件钱包支持**
    - **当前状态**：网络上线初期暂不支持
 
 
  - **关键提示**：务必保存并安全保管您的助记词，这是最终的恢复手段

3. **机器学习操作密钥管理**
     - 必须使用 `file` 密钥环后端，以便在服务器上实现程序化访问的加密存储
 
 
   - 在服务器上以加密形式存储，并使用强密码保护
 
 
   - 定期通过账户密钥授权轮换机器学习操作密钥
 
 
   - 在保持静态加密的同时，允许容器进行程序化访问

4. **运行安全**
     - 为所有密钥制定适当的备份和恢复流程
 
 
   - 在生产部署前，在安全环境中测试密钥恢复流程
 
 
   - 监控并审计密钥使用行为

### 恢复流程
1. **账户密钥丢失**：**关键** - 若无助记词则无法恢复
2. **机器学习操作密钥丢失**：创建新密钥，并通过账户密钥重新授权
3. **共识密钥丢失**：通过账户密钥授权轮换共识密钥
