# 提案

在 Gonka 中，有两种主要方式来提出和协调变更：治理提案和改进提案。

## 改进提案（链下）

用于讨论长期规划、重大架构设想以及制定社区发展路线图。它们类似于比特币的 [BIPs](https://github.com/bitcoin/bips)。

- 以 Markdown 文件形式管理，存放于 [/proposals](https://github.com/gonka-ai/gonka/tree/main/proposals) 目录中  
- 任何人都可以通过提交 Pull Request 提出新提案  
- 活跃参与者在 GitHub 上评审提案  
- 若提案获得批准，PR 将被合并到主仓库中  

有关链下改进提案，请参阅 [/proposals](https://github.com/gonka-ai/gonka/tree/main/proposals) 文件夹。

## 治理提案（链上）

用于对直接影响网络且需要链上投票的变更：

- 修改网络参数（例如通过 `MsgUpdateParams`）  
- 执行软件升级  
- 引入新模型  
- 引入新功能  
- 任何其他必须由社区在链上批准的修改  

治理权力来源于可验证的计算工作，而非被动持有代币。默认情况下，每个节点（Host）通过 PoC 获得的投票权重中，仅有 20% 自动激活。要解锁剩余的 80%，节点必须质押 GNK 代币作为抵押品，从而将治理影响力与真实的经济承诺绑定。技术细节（包括权重激活机制和抵押比例）详见 [Gonka: 代币经济模型](https://gonka.ai/tokenomics.pdf)。

!!! note "过渡期"
    在前 180 个周期（约 6 个月）内，新参与者仅需通过 PoC 即可参与治理并获得投票权重，无需抵押要求。在此期间，完整的治理权利可用，但投票权重仍与已验证的计算活动挂钩。
