# 如何编辑验证者公开信息

本指南介绍如何更新你的验证者配置文件，包括人类可读的名称、网站和头像/个人资料身份，以便浏览器显示正确信息。

## 先决条件

- 你必须是验证者的操作员（你持有操作员密钥）。
- 你的节点必须正在运行并连接到网络。
- 如果你想要经过验证的头像，请拥有身份服务（例如，[Keybase](https://keybase.io/)）。

## 字段/参数

以下是你可以设置或编辑的**唯一**字段。

| 字段    | 标志       | 用途/显示内容                                                                 |
|----------|------------|----------------------------------------------------------------------------------------------|
| Moniker  | `--new-moniker`  | 你的验证者的公开名称，在浏览器中显示。                  |
| Website  | `--website`  | 指向你的验证者网站或项目页面的链接。显示以便委托者了解更多。    |
| Identity | `--identity` | 通常用于提供验证/证明身份（例如，[Keybase](https://keybase.io/)，许多浏览器使用它来获取你的头像/徽标。你需要从网站下载应用程序以生成 PRP 密钥来获取你的徽标。 |

## 分步指南
如果你还没有 PGP 密钥，请运行以下命令。
```
keybase pgp gen
```

??? note "使用 Keybase 生成 PGP 密钥（keybase pgp gen）"
    `keybase pgp gen` 为此账户生成新的 PGP 密钥。在所有情况下，它使用现有设备密钥对公钥进行签名，并将签名推送到服务器。因此，用户运行此操作后将拥有公开可见的"PGP 设备"。PGP 密钥的秘密部分默认写入用户的本地 Keybase 密钥链，并使用"本地密钥安全"（LKS）协议加密。（更多信息，请尝试 keybase
